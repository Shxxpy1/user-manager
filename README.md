# Управление пользователями — Angular 20

Приложение для управления пользователями через JSONPlaceholder API, стилизованное с помощью NG-ZORRO Ant Design.

## 🚀 Демо

**[https://ВАШ_ЛОГИН.github.io/user-manager/](https://ВАШ_ЛОГИН.github.io/user-manager/)**

> Замените ссылку на реальную после деплоя.

## ✨ Функциональность

- **Список пользователей** — таблица с пагинацией (5/10/20 на странице) и поиском по имени или email
- **Карточка пользователя** — подробная информация с разделами «Контакты» и «Адрес»
- **Создание / Редактирование** — Reactive Forms с валидацией (обязательные поля, формат email)
- **Удаление** — с диалогом подтверждения, индикатор загрузки на кнопке
- **Тёмная / светлая тема** — переключение с анимацией, тема сохраняется между сессиями
- **Skeleton-загрузка** — вместо спиннера при загрузке данных
- **Обработка ошибок** — детальные сообщения + кнопка «Повторить попытку»
- **Страница 404** — для несуществующих маршрутов
- **NG-ZORRO UI** — компоненты Ant Design
- **Маршрутизация** — `/users`, `/users/:id`, `/users/new`, `/users/:id/edit`

## ⚠️ Важно про API

JSONPlaceholder — учебный mock-API. Он принимает запросы на создание/обновление/удаление и возвращает успешный ответ, **но данные не сохраняет между запросами**. Это ограничение самого API, указанного в задании. Список пользователей дополнен локальными данными (25 записей) для демонстрации пагинации.

## 📦 Запуск локально

### Требования
- Node.js ≥ 18
- Angular CLI ≥ 20

### Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/ВАШ_ЛОГИН/user-manager.git
cd user-manager

# Установить зависимости
npm install

# Запустить dev-сервер
ng serve
```

Откройте [http://localhost:4200](http://localhost:4200)

### Сборка для продакшена

```bash
ng build --configuration=production
```

## 🚢 Деплой на GitHub Pages

```bash
# 1. Установить инструмент деплоя (один раз)
npm install -g angular-cli-ghpages

# 2. Собрать с указанием имени репозитория
ng build --configuration=production --base-href="/user-manager/"

# 3. Задеплоить (автоматически создаст ветку gh-pages)
npx angular-cli-ghpages --dir=dist/user-manager/browser
```

После этого в настройках репозитория (Settings → Pages) выберите ветку `gh-pages`.
